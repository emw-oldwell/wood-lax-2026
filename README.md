# HHH Philly Girls Lacrosse — Summer 2026 Events

A phone-first dashboard for tracking Kate &amp; Rachel's HHH Philly 2028 / Lake Norman summer schedule. Tournaments, lodging, drive times, contact info, and document attachments, all in one tappable spot.

## Live URL

After deploying: `https://wood-lax-2026.vercel.app`

Share that link with Jen, the girls, grandparents, coaches — anyone who needs the schedule.

## Deploy to Vercel (first time)

### Option A — GitHub auto-deploy (recommended)

1. Create a new GitHub repo named `wood-lax-2026`.
2. From this folder:
   ```bash
   git init
   git add .
   git commit -m "Initial schedule site"
   git branch -M main
   git remote add origin https://github.com/<you>/wood-lax-2026.git
   git push -u origin main
   ```
3. In Vercel dashboard → **Add New… → Project** → Import the GitHub repo.
4. Framework preset: **Other**. Build command: `npm run build`. Output directory: leave blank (it serves the root).
5. **Deploy**. In **Settings → Domains** rename the alias to `wood-lax-2026.vercel.app`.

From then on, every `git push` auto-deploys in ~10 seconds.

### Option B — Vercel CLI

```bash
npm i -g vercel
cd wood-lax-2026
vercel              # first run prompts for project setup
vercel --prod       # promote to production
```

## How drag-and-drop docs work

This is the magic part. **You never need to edit `data.js` to add a document.**

1. Drag any file (PDF, photo, Word doc, Excel, etc.) into the appropriate `/docs/<category>/` folder.
2. Commit + push. (Or drag straight into the GitHub web UI: navigate to the folder → **Add file → Upload files**.)
3. Vercel runs `npm run build` automatically on every deploy. `build.js` scans `/docs/` and writes `docs.json`.
4. The Docs tab on the live site fetches `docs.json` and shows the new file.

Filename → display name conversion:
- `philly-phinest-info.pdf` → "Philly Phinest Info"
- `Goaliesmith Schedule 2026.pdf` → "Goaliesmith Schedule 2026"
- (file extension and size show as a sub-label)

Folder names become category headers. Add new categories by creating new folders in `/docs/`.

## Updating tournaments / lodging

Edit `data.js`:
- **New tournament**: append to `events` array.
- **Hotel changed**: edit the matching object in `lodging`.
- **Lodging by night**: edit the `lodgingByDate` builder.
- **Practices added/removed**: edit `practices` array.

Save → commit → push (or `vercel --prod`).

## What's in the dashboard

Six tabs accessible from the sticky top nav:

1. **Upcoming** — next 5 events + season-at-a-glance stats
2. **Calendar** — June + July grids, color-coded by where you sleep that night
3. **Lodging** — chronological cards, every conf #/phone/address tappable
4. **Tournaments** — full list with venue map links and drive times from each event's actual lodging
5. **Docs** — auto-populated from `/docs/` folder
6. **Players** — Kate and Rachel cards

## Adding to a phone home screen

iOS: Open in Safari → Share → **Add to Home Screen**. Installs with the page icon and runs full-screen like a native app.

Android: Open in Chrome → menu → **Add to Home screen**.

## Privacy note

The Vercel URL is public — anyone with the link can view. There's no login. If you want to add a password gate later, Vercel supports it via the dashboard (Settings → Deployment Protection).
